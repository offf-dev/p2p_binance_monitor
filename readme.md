1. manifest.json
   Назначение: Это основной файл конфигурации расширения, который определяет его структуру, права доступа, подключаемые скрипты, стили и другие ресурсы. Без него Chrome не сможет загрузить расширение.

Описание:

"manifest_version": 3: Указывает версию стандарта Chrome Extensions (Manifest V3 — актуальный стандарт на 2025 год).
"name", "version", "description": Метаданные расширения (название, версия, описание).
"permissions": Права, необходимые расширению:
activeTab: Доступ к активной вкладке.
scripting: Возможность выполнять скрипты на странице.
storage: Для сохранения настроек (курс и сумма) между сеансами.
"content_scripts": Указывает, что content.js и styles.css будут применяться к страницам, соответствующим шаблону p2p.binance.com/*.
"icons": Ссылки на иконки расширения, которые отображаются в интерфейсе Chrome.
Роль: Этот файл сообщает Chrome, как загрузить и запустить расширение, какие файлы использовать и на каких сайтах оно активно.

2. content.js
   Назначение: Основной JavaScript-файл, который выполняется на странице p2p.binance.com. Он создает панель управления, мониторит таблицу, проверяет данные и воспроизводит звуковой сигнал при совпадении.

Описание:

Создание панели:
Создается <div> с id="p2p-monitor-panel", содержащий два поля ввода (targetRate и targetAmount), кнопки "Start" и "Stop", и элемент для статуса.
Панель добавляется в начало <body> страницы с помощью document.body.prepend(panel).
Звук:
Создается объект Audio с URL на звуковой файл (в примере — beep-01a.mp3). Вы можете заменить его на локальный файл, добавив его в папку расширения, например, audio/beep.mp3.
Сохранение и загрузка настроек:
saveSettings(): Сохраняет введенные курс и сумму в chrome.storage.local.
loadSettings(): Загружает сохраненные значения при загрузке страницы.
Мониторинг таблицы:
checkTable(): Каждую секунду проверяет первые 5 строк таблицы (селектор .css-1k1p5jm — замените на актуальный).
Извлекает цену (селектор .css-1r81yi4) и доступную сумму (селектор .css-1e4w52c), обрабатывая текст для получения чисел.
Если цена ≤ заданного курса и доступная сумма ≥ заданной суммы, воспроизводится звук, а статус обновляется зеленым цветом с информацией о совпадении.
Если совпадений нет, статус становится красным с текстом "No matches".
Управление мониторингом:
Кнопка "Start" запускает интервал (setInterval) для вызова checkTable каждую секунду.
Кнопка "Stop" останавливает интервал (clearInterval) и сбрасывает статус.
Инициализация: При загрузке страницы вызывается loadSettings() для восстановления сохраненных значений.
Важно: Селекторы (.css-1k1p5jm, .css-1r81yi4, .css-1e4w52c) являются предположительными. Вам нужно открыть DevTools (F12) на странице p2p.binance.com, найти таблицу с мейкерами и заменить селекторы на актуальные классы или атрибуты.

3. styles.css
   Назначение: Файл стилей для оформления панели управления, чтобы она выглядела аккуратно и не мешала содержимому страницы.

Описание:

#p2p-monitor-panel:
Панель фиксируется в левом верхнем углу (position: fixed; top: 10px; left: 10px).
Фон светло-серый (#f0f0f0), с рамкой и скругленными углами для аккуратного вида.
z-index: 1000 обеспечивает отображение поверх контента сайта.
display: flex; gap: 10px выстраивает элементы (поля, кнопки, статус) в ряд с промежутками.
input: Поля ввода имеют фиксированную ширину (100px) и отступы для удобства.
button: Кнопки с отступами и указателем курсора для интерактивности.
#status: Статус отображается жирным шрифтом для выделения.
Роль: Делает панель удобной и визуально приятной, не перекрывая важные элементы сайта.

4. icon48.png и icon128.png
   Назначение: Иконки расширения, которые отображаются в интерфейсе Chrome (например, в списке расширений или на панели инструментов).

Описание:

icon48.png: Иконка размером 48x48 пикселей.
icon128.png: Иконка размером 128x128 пикселей.
Формат должен быть PNG.
В артефакте указаны как заглушки ("base64 encoded icon data"). Вы можете:
Создать свои иконки (например, с логотипом или символом, связанным с Binance).
Использовать любые PNG-изображения подходящего размера.
Конвертировать изображения в base64 и добавить в manifest.json или просто поместить файлы в папку расширения.
Как создать:

Используйте графический редактор (Photoshop, GIMP, Canva) для создания изображений 48x48 и 128x128.
Сохраните как icon48.png и icon128.png и поместите в папку расширения.
Роль: Обеспечивают визуальную идентификацию расширения в браузере.

Как это работает вместе
Загрузка расширения:
Chrome читает manifest.json, чтобы понять, какие файлы использовать и где применять расширение.
При открытии страницы p2p.binance.com подключаются content.js и styles.css.
Инициализация:
content.js создает панель управления, добавляет ее на страницу и загружает сохраненные настройки (курс и сумма) из chrome.storage.local.
styles.css оформляет панель, делая ее фиксированной вверху страницы.
Мониторинг:
Пользователь вводит желаемый курс и сумму в поля панели.
При нажатии "Start" запускается проверка таблицы каждую секунду.
Если курс и сумма у одного из первых 5 мейкеров соответствуют заданным, воспроизводится звук, а статус обновляется.
Кнопка "Stop" останавливает мониторинг.
Иконки:
icon48.png и icon128.png отображаются в интерфейсе Chrome для идентификации расширения.
Как установить
Создайте папку, например, p2p_binance_monitor.
Создайте файлы:
manifest.json с указанным содержимым.
content.js с кодом JavaScript.
styles.css с указанными стилями.
Добавьте icon48.png и icon128.png (создайте или скачайте изображения 48x48 и 128x128).
Откройте Chrome, перейдите в chrome://extensions/.
Включите "Режим разработчика" (вверху справа).
Нажмите "Загрузить распакованное расширение" и выберите папку.
Откройте p2p.binance.com, и панель появится вверху.
